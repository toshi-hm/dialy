# 受け入れテストケース（FR-01〜FR-05）

最終更新: 2026-02-28

## 目的

MVP機能要件（FR-01〜FR-05）に対する受け入れテストケースを定義し、品質基準を明確化する。

## FR-01: 日付選択機能

| ID | テストケース | 期待結果 | 検証方法 |
|---|---|---|---|
| AT-01-01 | 初回表示時のデフォルト日付 | 今日の日付が「◯月◯日（曜日）」形式で表示される | Unit / E2E |
| AT-01-02 | Dial時計回り回転 | 翌日（未来日付でない場合）に切り替わる | Unit |
| AT-01-03 | Dial反時計回り回転 | 前日に切り替わる | Unit |
| AT-01-04 | 未来日付選択の禁止 | 今日より先の日付が選択できず、エラーメッセージが表示される | Unit / E2E |
| AT-01-05 | Dial操作レスポンス | 100ms以内に日付が更新される | Performance |
| AT-01-06 | カレンダーダイアログ表示 | Dialクリックでカレンダーが開く | E2E |
| AT-01-07 | カレンダーから日付選択 | 選択した日付に切り替わり、カレンダーが閉じる | Unit |
| AT-01-08 | カレンダーの未来日付無効化 | 未来日付がグレーアウトされ選択不可 | Unit |

## FR-02: 日記作成機能

| ID | テストケース | 期待結果 | 検証方法 |
|---|---|---|---|
| AT-02-01 | 新規日付でのテキスト入力 | テキストエリアが空で表示される | Unit / E2E |
| AT-02-02 | 1秒間入力停止で自動保存 | 保存ステータスが「保存中...」→「✓ 保存しました」に遷移 | Unit / E2E |
| AT-02-03 | 保存データの永続化 | ページリロード後も同じ日付で内容が復元される | E2E |
| AT-02-04 | 文字数カウント表示 | 入力文字数がリアルタイムで表示される | Unit / E2E |
| AT-02-05 | 10,000文字上限の入力制御 | 10,000文字を超える入力が制限される | Unit |
| AT-02-06 | 未来日付での作成禁止 | 未来日付では日記が作成されない | Unit |
| AT-02-07 | 同日重複作成の防止 | 同じ日に2つ目のエントリーを作成しようとすると既存を編集モードにする | Unit |
| AT-02-08 | 保存失敗時のリトライ | 最大3回の指数バックオフリトライ後に失敗メッセージが表示される | Unit |

## FR-03: 日記編集機能

| ID | テストケース | 期待結果 | 検証方法 |
|---|---|---|---|
| AT-03-01 | 既存日記の自動読み込み | 日記が存在する日付を選択すると、内容がテキストエリアに表示される | Unit / E2E |
| AT-03-02 | 編集内容の自動保存 | 内容変更後1秒で自動保存される | Unit / E2E |
| AT-03-03 | updatedAt の更新 | 保存時にupdatedAtが現在時刻に更新される | Unit |
| AT-03-04 | 最新版のみ保持 | 編集履歴は保持されず、最新版のみ保存される | Unit |
| AT-03-05 | 10,000文字制限の維持 | 編集時も文字数上限が適用される | Unit |

## FR-04: 過去同日日記表示機能

| ID | テストケース | 期待結果 | 検証方法 |
|---|---|---|---|
| AT-04-01 | 過去5年分の同月日表示 | 選択日と同じ月日の過去エントリーが最大5年分表示される | Unit / E2E |
| AT-04-02 | 新しい順のソート | 直近年度が先頭に表示される | Unit |
| AT-04-03 | プレビューカード情報 | 年度・約100文字プレビュー・文字数が表示される | Unit |
| AT-04-04 | カード展開/折りたたみ | クリック/タップで全文表示に展開、再度クリックで折りたたみ | Unit / E2E |
| AT-04-05 | 過去日記なしの表示 | 過去の同日日記がない場合「過去の日記はまだありません」と表示 | Unit / E2E |
| AT-04-06 | エントリーなし年の非表示 | 日記が存在しない年度はリストに表示されない | Unit |

## FR-05: 日記削除機能

| ID | テストケース | 期待結果 | 検証方法 |
|---|---|---|---|
| AT-05-01 | 削除ボタンの表示条件 | 編集中の日記がある場合のみ削除ボタンが表示される | Unit / E2E |
| AT-05-02 | 確認ダイアログの表示 | 削除ボタン押下時に確認ダイアログが表示される | Unit / E2E |
| AT-05-03 | 確認後の削除実行 | 確認ダイアログで「削除」を選択すると日記が削除される | Unit / E2E |
| AT-05-04 | キャンセルでの保持 | 確認ダイアログで「キャンセル」を選択すると日記が保持される | Unit / E2E |
| AT-05-05 | 削除後のUI更新 | 削除後テキストエリアがクリアされ、過去同日リストも更新される | Unit / E2E |
| AT-05-06 | 削除の不可逆性 | 削除された日記は復元できない | Unit |

## 非機能要件チェック

| ID | テストケース | 基準 | 検証方法 |
|---|---|---|---|
| AT-NF-01 | 初回表示速度 | 3秒以内 | Lighthouse |
| AT-NF-02 | Dial操作応答 | 100ms以内 / 60fps | Performance計測 |
| AT-NF-03 | キーボードナビゲーション | Tab移動、Enter/Space操作が可能 | Manual / E2E |
| AT-NF-04 | スクリーンリーダー対応 | ARIA属性による適切な読み上げ | Manual / a11y audit |
| AT-NF-05 | レスポンシブ表示 | モバイル/タブレット/デスクトップで適切に表示 | E2E / Manual |
| AT-NF-06 | コントラスト比 | WCAG 2.1 AA（テキスト4.5:1、大文字3:1） | Storybook a11y |
